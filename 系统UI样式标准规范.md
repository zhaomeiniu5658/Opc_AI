版本：2026-07-10  
状态：已按当前前端实现同步  
适用范围：登录页、应用平台主界面、视频/图片/音频/解析/设置/帮助/2D数字人模块、Agent 应用、管理平台、顶部导航栏、左侧菜单栏、用户菜单、历史记录与执行过程面板。

本文档以当前代码为准同步，主要参考：

- `platform-frontend/platform-index.html`
- `platform-frontend/assets/app.css`
- `platform-frontend/assets/agent-app.css`
- `platform-frontend/assets/user-admin.css`
- `platform-frontend/assets/auth-login.css`

重要备注：新增页面、模块、控件时必须优先使用本文档规范。若规范未覆盖具体业务场景，可以扩展新样式，但必须保持现有深色工作台体系、控件密度、圆角、间距和交互状态一致。

## 1. 设计原则

- 系统整体是深色工作台，不做浅色大面积背景。
- 功能页面第一屏直接呈现可操作内容，不做营销式落地页。
- 页面信息密度偏工作台化，避免过大的装饰性卡片。
- 主要容器采用深色面板、细边框、轻阴影或无阴影；模块内部容器优先使用 `8px` 圆角。
- Agent 应用允许使用更强的紫色激活态，但仍保持深色控制台风格。
- 操作按钮优先使用图标或图标加文字；低频动作不要挤占标题栏。
- 所有文本必须不溢出、不重叠、不遮挡按钮；长文本使用省略、滚动或自然换行。
- 修改 CSS 后必须同步 HTML 资源版本号，避免浏览器缓存旧样式。

## 15. 响应式规则

- 小屏下应用模块双列布局可以改为单列。
- `audio-layout`、`parse-layout` 在 `max-width: 1100px` 下改为单列。
- `720px` 以下，标题栏操作区可以换行，表单网格改为单列。
- 侧栏在小屏可以调整宽度或转为上方布局，但菜单项文本不能挤压重叠。
- 历史记录操作按钮优先保留 `32px` 图标按钮尺寸。
- Agent 工作台移动端需要降低面板内边距，日志抽屉不得遮挡核心输入区。
- 管理平台统计卡片在 `920px` 以下从 3 列降为 2 列，`720px` 以下降为 1 列。

## 16. 实施约束

- 样式调整优先修改 CSS，不改变业务数据结构。
- 应用平台样式主文件为 `app.css`；Agent 专用样式在 `agent-app.css`；管理平台样式在 `user-admin.css`；登录页样式在 `auth-login.css`。
- 不要在新模块中复制大段旧 CSS；优先复用现有类名和结构。
- 新增按钮、菜单、卡片时优先使用 `8px` 圆角，除头像、状态胶囊、滚动条等自然需要圆形的元素。
- 所有中文内容文件保持 UTF-8；CSS `content` 中的中文建议使用 Unicode 转义。
- 修改 CSS 或 HTML 引用资源后，更新 `platform-index.html` 中对应资源版本号。
- 文档与实现不一致时，以当前 CSS 为准，并同步更新本文档。
